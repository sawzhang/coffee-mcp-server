Why Coffee MCP?

When users ask AI for coffee, only brands with MCP are reachable.

For Brands

Connect to every AI assistant in 2-3 weeks. Zero MCP knowledge needed. Choose your category preset (coffee/tea/juice/bakery), configure a YAML file, implement your API adapter.

For Consumers

Say "order me an oat latte" to any AI. It finds your nearest store, checks coupons, calculates price, and creates the order — all through natural conversation.

Security: L0-L3 Risk Levels

Every tool is classified by risk. Higher risk = stricter controls.

LevelRate LimitProtectionExample Tools
L0 Public Read 60/min per IPNone needed browse_menu, nearby_stores, nutrition_info
L1 Auth Read 30/min per userOAuth token my_account, my_coupons, my_orders
L2 Auth Write 5/hour per userRate limit claim_all_coupons, create_address
L3 High Risk 10/day per userConfirmation token + Idempotency key create_order, stars_redeem

Brand Onboarding

From zero to AI-discoverable in three steps

1. brand-init (5 min)

uv run brand-init

Choose category preset (coffee/tea/juice/bakery). Answer 3 questions. Generates brand.yaml.

2. brand-onboard (30 min)

/brand-onboard <api-docs>

AI reads your API docs, maps 21 methods, generates adapter.py + tests.

3. Deploy & Test

BRAND=xxx uv run coffee-company-toc

Start MCP server. Connect AI assistants. Verify end-to-end.

Zero code needed for step 1. The platform handles security (L0-L3 rate limiting, confirmation tokens, idempotency), PII masking, parameter validation, and markdown formatting. Your adapter only maps API fields.

Quick Start

# Install $ git clone https://github.com/sawzhang/coffee-mcp.git $ cd coffee-mcp && uv sync # Run ToC server (consumer ordering) $ uv run coffee-company-toc # Run with HTTP transport (for OpenClaw / remote clients) $ uv run coffee-company-toc-http # Initialize a new brand $ uv run brand-init # Run all tests (122 total) $ uv run python tests/test_toc_mcp.py # 89 passed $ uv run python tests/test_mcp_real.py # 33 passed
