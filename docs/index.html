Starbucks MCP Server

星巴克中国 B2B 开放平台 MCP 接入层 —— 让 AI Agent 通过 MCP 协议调用现有 HTTP 开放平台全部能力

MCP Protocol Kong Gateway 10 Tools (Phase 1) B2B Zero Platform Changes

Architecture

B2B Agent (NIO / Qwen / Fliggy / Custom) | MCP Protocol (SSE / Streamable HTTP) v +-------------- Kong (existing) ---------------+ | | | HMAC Auth IP Whitelist Rate Limit ACL | <-- reuse existing B2B credentials | | | /sse, /mcp --> MCP Adapter (new service) | <-- 2 new routes | /coupon/* --> openapi-platform (unchanged) | | /cashier/* --> openapi-platform (unchanged) | +-------------------+--+------------------------+ | | +-------------+ +------------+ v v MCP Adapter openapi-platform (this project) (zero changes) - MCP protocol - existing HTTP APIs - tool registry - business logic - semantic formatting - data layer - permission filtering | | internal network v openapi-platform backend

Kong handles auth/rate-limiting/ACL. MCP Adapter only does protocol conversion + semantic formatting.

Phase 1 Tools (Read-Only)

member_query
POST /crmadapter/account/query
Query member info by mobile, openId, or sbuxId
member_tier
POST /crmadapter/account/memberTier
Member tier details: stars, expiry, next-tier gap
member_benefits
POST /crmadapter/customers/getBenefits
8 benefit statuses: welcome, birthday, upgrade rewards
member_benefit_list
POST /crmadapter/asset/coupon/getBenefitList
List all coupons and benefit vouchers for a member
coupon_query
POST /coupon/query
Check if coupons were generated for an order
coupon_detail
POST /coupon/detail
Coupon details: status, face value, redemption count
equity_query
POST /equity/query
Check equity/benefit distribution status
equity_detail
POST /equity/detail
Full equity details: amount, validity, redemption
assets_list
POST /assets/list
All customer assets: UPP coupons + benefit coupons
cashier_pay_query
POST /cashier/payQuery
Payment status: in-progress / success / failed

Quick Start

1

Clone & Install

Requires uv + Python 3.13

$ git clone https://github.com/sawzhang/starbucks-mcp-server.git $ cd starbucks-mcp-server && uv sync
2

Run the Demo

9-step B2B scenario: member -> tier -> benefits -> coupons -> equity -> assets -> payment

$ uv run sbux demo # Step 1: member_query(mobile="138****1234") # Step 2: member_tier(sbux_id="SBUX_M_100001") # Step 3: member_benefits(sbux_id="SBUX_M_100001") # ... through Step 9: cashier_pay_query
3

Connect AI Agent

Add to Claude Code or Cursor MCP config

{ "mcpServers": { "starbucks": { "command": "uv", "args": ["--directory", "/path/to/starbucks-mcp-server", "run", "starbucks-mcp"] } } }

B2B Scenario: NIO Car Agent

NIO Car Owner:
"Check my Starbucks membership and available coupons before I arrive"

NIO Agent (via MCP):

Step 1: member_query(mobile="138****1234") → Gold member, 142 stars
Step 2: member_tier(sbux_id="SBUX_M_100001") → 358 stars to Diamond
Step 3: assets_list(sbux_id="SBUX_M_100001") → 3 available coupons
"You're a Gold member with 142 stars. You have 3 coupons available: a Tall drink voucher (expires Apr 30), a birthday free drink (expires Mar 31), and a friend referral reward worth ¥30."
